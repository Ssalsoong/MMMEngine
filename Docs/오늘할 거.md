

## SceneRef

- 슬롯 맵을 사용한 핸들
- slot idx, slot gen 으로 SceneManager의 Scene Table에 접근하여 기능을 쓸 수 있음





## Screen

- 엔진이 사용중인 화면 핸들에 대한 정보를 담은 객체 (정적임)
- 즉 실제 윈도우 핸들이 아닌 엔진이 개념적으로 인식하고 있는 윈도우 핸들임
- 후술할 Application에서 Screen의 정보를 실제 윈도우 핸들과 동기화해주는 로직이 필요함.



## Application (메인 Window 핸들) 

- 윈도우 루프 제공
- 내부 루프에서 돌릴 함수 Update에 사용자가 원하는 로직을 넣어 둘 수 있음
  - 에디터의 경우에는 에디터가 돌리는 윈도우들의 update로직과 가상엔진 즉 플레이 테스트를 위한 가상실행을 담당하는 로직을 넣어 둘 수 있음
  - 플레이어도 마찬가지이나 게임 실행에 필요한 것만(게임 수정이 아닌) 로직을 골라내서 처리하게 만들어 놓을 수 있음

## Serializable Event

- 이름 뜻대로 직렬화 가능한 이벤트
- 델리게이트인데 저장/불러오기 가능함
- 함수는 RTTR을 이용한 런타임 이름찾기로, 호출자 참조는 GUID로 복원
- Fast Delegate 내부구현을 감쌈
- 람다는 절대 등록할 수 없음 (람다가 필요하면 기존 델리게이트를 사용할 것)
